Home AI — описание проекта
Home AI — это персонализированное web-приложение-помощник для планирования и напоминания о рутинных домашних делах, построенное на основе распределённого REST API и нейросетевых технологий.
 
Для кого предназначено
Молодые люди 18–30 лет, которые впервые живут самостоятельно, работают или учатся и часто забывают о бытовых обязанностях.
 
Основные задачи, о которых напоминает приложение
•	уборка (полы, пыль, кухня, санузел);
•	стирка / глажка;
•	мытьё посуды (если нет посудомойки);
•	вынос мусора;
•	полив цветов;
•	закупка продуктов / контроль остатков;
•	уход за домашними животными;
•	замена фильтров, лампочек и другие мелкие бытовые дела.
 
Главные фишки приложения
1. Персонализация с первого запуска
Короткий опрос (6–10 вопросов) о наличии техники (посудомойка, робот-пылесос), количестве комнат, наличии растений и животных, предпочтительной частоте уборки. На основе ответов генерируется стартовый набор задач с приблизительной периодичностью.
2. Нейросетевая генерация подзадач
Пользователь может ввести сложную задачу (например, «генеральная уборка»), а система с помощью локальной языковой модели GPT4All автоматически разбивает её на несколько простых шагов.
Генерация происходит асинхронно в отдельном ML-микросервисе (FastAPI), который вызывается через очередь задач Celery. Это позволяет не блокировать основной бэкенд и демонстрирует распределённую архитектуру.
3. Эвристический анализ повторяющихся задач
На основе истории выполненных задач система выявляет те, которые пользователь выполняет с регулярной периодичностью (например, раз в неделю). При обнаружении такой закономерности приложение предлагает сохранить её как шаблон повторяющейся задачи, чтобы в будущем напоминания устанавливались автоматически.
4. Распределённая архитектура на основе REST API
•	Основной бэкенд реализован на Django + Django REST Framework (DRF) и предоставляет REST API для всех операций: управление пользователями, задачами, историей выполнения, опросом.
•	Аутентификация — JWT-токены (djangorestframework-simplejwt) в сочетании с сессиями для web-интерфейса.
•	Асинхронные задачи (генерация подзадач, анализ повторяющихся задач) выполняются с помощью Celery с брокером Redis.
•	ML-микросервис написан на FastAPI и отвечает исключительно за вызов нейросети GPT4All. Основной бэкенд взаимодействует с ним через HTTP.
•	База данных — PostgreSQL, обеспечивающая надёжное хранение и транзакционность.
•	Все компоненты упакованы в Docker-контейнеры и управляются через Docker Compose, что гарантирует идентичность окружения на всех этапах разработки и развёртывания.
5. Интерфейс и уведомления
•	Фронтенд реализован на Django Templates (SSR) — серверный рендеринг HTML. Это позволяет быстро получить работающий интерфейс без отдельной сборки React.
•	Основные экраны: календарь на неделю, список задач на сегодня с возможностью отметки выполнения, форма ручного добавления задач, панель управления уведомлениями.
•	Уведомления реализованы через WebSocket (Django Channels) для мгновенных оповещений, а также предусмотрена возможность добавления Push-уведомлений на последующих этапах развития проекта.
